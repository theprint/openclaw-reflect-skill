# Session Reflection Template

Use for `/reflect` — quick post-work review of the current session.

---

## Session: [date/time]

### What Was Done
List the main tasks, decisions, and outcomes from this session.

### Expectations vs Reality
- Did the work match what was requested?
- Any surprises (good or bad)?
- What would I do differently?

### Lessons
Any mistakes, workarounds, or discoveries worth remembering. Be specific — "X failed because Y" not "had some issues."

### User Observations
- Energy level / mood
- Working style patterns noticed
- Preferences expressed
- Frustration points or enthusiasm triggers

### Capture Checklist
- [ ] `Learnings.md` — any new lessons?
- [ ] `USER.md` — preference or context changes?
- [ ] `MEMORY.md` — significant events worth keeping long-term?
- [ ] `memory/YYYY-MM-DD.md` — raw daily notes updated?
- [ ] `HEARTBEAT.md` — any monitoring rule changes needed?

### Follow-Ups
- Tasks to create or update
- Things to check next session
- Open questions for the user

---

## What Makes a Good Session Reflection

**Do:**
- Be honest about what didn't work — this is how the skill improves
- Connect the lesson to a specific moment, not a vague feeling
- Check if this mistake has happened before (pattern > one-off)
- Keep it under 200 words total unless something significant happened

**Don't:**
- Manufacture content — if nothing significant happened, say so
- Write a progress report — Rasmus was there, he knows what happened
- Over-formalize — this is a quick checkpoint, not a document
- Apologize ritually — note the lesson and move on
